#+TITLE: lark.el — Emacs Interface to Lark/Feishu
#+AUTHOR: lark.el contributors
#+OPTIONS: toc:3

An Emacs package that wraps [[https://github.com/larksuite/cli][lark-cli]] to bring Lark/Feishu into Emacs.
Transient menus for command dispatch, mu4e-style mail, dired-like drive
browsing, org-table spreadsheets, and async execution throughout.

* Prerequisites

- Emacs 28.1+ (for built-in =transient=)
- [[https://github.com/larksuite/cli][lark-cli]] installed and on your =PATH=

** Installing lark-cli

#+begin_src shell
npm install -g @larksuite/cli
#+end_src

Verify it works:

#+begin_src shell
lark-cli --help
#+end_src

* Installation

** From source (straight.el)

#+begin_src emacs-lisp
(straight-use-package
 '(lark :type git :host github :repo "bbw9n/lark.el"))
#+end_src

** From source (manual)

Clone this repo and add it to your =load-path=:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/lark.el")
(require 'lark)
(lark-setup-prefix-key) ;; binds C-c l
#+end_src

** use-package

#+begin_src emacs-lisp
(use-package lark
  :load-path "/path/to/lark.el"
  :config
  (lark-setup-prefix-key))
#+end_src

** Doom Emacs

Add in packages.el from the source of github repo

  :recipe (:host github :repo "bbw9n/lark.el" :branch "main"))
#+end_src

* Quick Start: First-Time Setup

This section walks you through initial configuration, authentication,
and pulling up your calendar — all from within Emacs.

** Step 1: Configure lark-cli (one-time)

Before lark.el can talk to Lark, you need to register an app.
Run this in your terminal (or use =M-x shell=):

#+begin_src shell
lark-cli config init
#+end_src

This is an interactive prompt that asks for:

1. *Brand*: choose =feishu= (China) or =lark= (international)
2. *App ID & App Secret*: from the [[https://open.feishu.cn/app][Feishu Open Platform]] (or [[https://open.larksuite.com/app][Lark Developer Console]])
   - Go to the console, create an app (or use an existing one)
   - Copy the App ID and App Secret
3. The CLI stores credentials locally in =~/.config/lark-cli/=

If you prefer non-interactive setup:

#+begin_src shell
echo "YOUR_APP_SECRET" | lark-cli config init \
  --app-id cli_xxxxxxxxxxxx \
  --app-secret-stdin \
  --brand feishu
#+end_src

Verify:

#+begin_src shell
lark-cli config show
#+end_src

** Step 2: Authenticate (OAuth login)

Now authenticate your Lark user account. You can do this either from the
terminal or from Emacs:

*** Option A: From Emacs

#+begin_src
M-x lark-auth-login
#+end_src

Or use the keybinding: =C-c l l= then press =L= (Login).

This will:
1. Call =lark-cli auth login --no-wait= in the background
2. Open your browser to the Lark authorization page
3. Poll automatically until you approve in the browser
4. Show "Lark: login successful!" in the echo area

*** Option B: From terminal first

#+begin_src shell
lark-cli auth login --domain calendar
#+end_src

The =--domain= flag requests scopes for a specific domain. Available
domains: =calendar=, =task=, =im=, =mail=, =contact=, =docs=, =drive=,
=base=, =sheets=, =wiki=, =vc=, =approval=, =event=, =minutes=, or =all=.

You can request multiple domains:

#+begin_src shell
lark-cli auth login --domain calendar,task,im
#+end_src

Or request everything:

#+begin_src shell
lark-cli auth login --domain all
#+end_src

*** Verify authentication

From Emacs:

#+begin_src
M-x lark-auth-status
#+end_src

Or: =C-c l s=

This shows your login status in the echo area. You should see something
like "Lark: authenticated as Your Name (your-tenant)".

From terminal:

#+begin_src shell
lark-cli auth status
#+end_src

* AI Layer

lark.el includes an AI-native orchestration layer that turns natural-language
prompts into multi-step lark-cli execution plans.  Instead of remembering
which commands to chain together, you describe what you want and the AI
figures out the steps.

** How it works

1. You type a prompt (e.g., "what's on my plate today?")
2. lark.el selects the relevant lark-cli skill files (=SKILL.md=) based on
   keywords in your prompt
3. The skills + your prompt are sent to an LLM, which returns a JSON
   execution plan
4. The plan is shown in a review buffer — you can inspect, remove steps,
   or cancel
5. On confirmation, lark.el executes the CLI commands (in parallel where
   possible), collecting results
6. If the plan includes a synthesis step, results are sent back to the LLM
   for a human-readable summary
7. The final output is rendered in a buffer

Write operations (create, update, delete, send) are flagged in the plan and
require explicit confirmation before execution.

** Prerequisites

- An LLM backend: [[https://github.com/karthink/gptel][gptel]] (recommended) or any OpenAI-compatible HTTP endpoint
- lark-cli skills installed in =~/.agent/skills/=:

#+begin_src shell
npx skills add larksuite/cli -g -y
#+end_src

** Setup

*** With gptel (recommended)

#+begin_src emacs-lisp
(use-package gptel
  :config
  ;; configure your preferred provider
  (setq gptel-model 'claude-sonnet-4-20250514))

(setq lark-ai-backend 'gptel)  ;; this is the default
#+end_src

*** With a raw HTTP endpoint

#+begin_src emacs-lisp
(setq lark-ai-backend 'http
      lark-ai-http-endpoint "https://api.openai.com/v1/chat/completions"
      lark-ai-http-api-key "sk-...")
#+end_src

Or set the =LARK_AI_API_KEY= environment variable instead of hardcoding the key.

** Examples

*** Standup report

#+begin_src
C-c l i  "what's on my plate today?"
#+end_src

The AI generates a plan:

#+begin_example
Lark AI -- Execution Plan
----------------------------------------

  [0] Fetch today's agenda  (read-only)  [group 1]
      lark-cli calendar +agenda
  [1] Fetch open tasks  (read-only)  [group 1]
      lark-cli task +get-my-tasks
  [2] Summarize schedule and tasks  (AI synthesis)

----------------------------------------
  RET Execute  x Remove step  q Cancel
#+end_example

Steps 0 and 1 run in parallel (same group).  After execution, the LLM
synthesizes results into:

#+begin_example
## 2026-04-13 Sunday

### Agenda
| Time        | Event              | Organizer | Status   |
|-------------+--------------------+-----------+----------|
| 09:00-09:30 | Daily Standup      | Alice     | accepted |
| 14:00-15:00 | 1:1 with Manager   | Carol     | tentative|

### Open Tasks
- [ ] Review PR #142 (due: today)
- [ ] Update API docs (due: Apr 14)

### Notes
- No scheduling conflicts
- Free slots: 10:00-14:00, 15:00-18:00
#+end_example

Or use the dedicated command: =M-x lark-ai-workflow-standup=.

*** Meeting summary

#+begin_src
C-c l i  "summarise my meetings this week"
#+end_src

The AI chains =vc +search= -> =vc +notes= -> synthesis, producing a
structured report of all meetings with their notes and action items.

Or: =M-x lark-ai-workflow-meeting-summary=.

*** Schedule a meeting

#+begin_src
C-c l i  "set up a 30min sync with Alice next Tuesday afternoon"
#+end_src

The AI plan:

#+begin_example
  [0] Search contact "Alice"  (read-only)
  [1] Find available time slots  (read-only)
  [2] Check room availability  (read-only)
  [3] Create calendar event  CAUTION writes
#+end_example

Step 3 is marked as a side-effect and will prompt for confirmation
before executing.

Or: =M-x lark-ai-workflow-schedule=.

*** Context-aware actions

When you're in a Lark buffer (agenda, chat, task list, etc.), press
=C-c l .= to perform an AI action on the item at point.  The AI
automatically knows the domain and which item you're looking at.

For example, in the =*Lark Agenda*= buffer with point on an event:

#+begin_src
C-c l .  "reschedule to tomorrow at 3pm"
#+end_src

AI: Generated a plan. Review it in the plan buffer.
(after execution)
AI: You have 3 meetings tomorrow:
- 09:00-09:30  Daily Standup
- 11:00-12:00  Design Review
- 15:00-16:00  Sprint Retro

* Troubleshooting

** "Cannot find lark-cli executable"

lark-cli is not on your =PATH=. Either:

- Install it: =npm install -g @larksuite/cli=
- Set the full path: =(setq lark-cli-executable "/path/to/lark-cli")=

** "Lark: not authenticated"

Run =M-x lark-auth-login= or authenticate from terminal:

#+begin_src shell
lark-cli auth login --domain all
#+end_src

** Empty agenda / "no events found"

1. Check you have calendar scopes: =lark-cli auth check --scope calendar:readonly=
2. Try from terminal first: =lark-cli calendar +agenda=
3. If the terminal works but Emacs doesn't, enable debug logging:
   =(setq lark--debug t)= and check the =*lark-log*= buffer

** Permission errors

Some operations require specific scopes. Re-login with the needed domain:

#+begin_src shell
lark-cli auth login --domain calendar,task
#+end_src

** Debugging

Enable debug logging to see all CLI invocations:

#+begin_src emacs-lisp
(setq lark--debug t)
#+end_src

Then check =*lark-log*= buffer (=C-x b *lark-log*=) to see every command
that lark.el runs, including arguments and exit codes.

* Architecture

#+begin_example
lark.el                         Entry point, defgroup, prefix keymap, load-path setup

core/
  lark-core.el                  Process management, JSON parsing, CLI debug line stripping
  lark-auth.el                  OAuth device-code login, status, identity switching
  lark-transient.el             Top-level transient dispatch menu

domain/
  lark-calendar.el              Calendar: agenda, events, create, free/busy, RSVP
  lark-tasks.el                 Tasks: list, create, complete, subtasks, tasklists
  lark-im.el                    Messenger: chats, messages, send, reply, events
  lark-mail.el                  Mail: mu4e-style inbox, shr HTML rendering, compose
  lark-docs.el                  Docs: search, fetch, create, update, media operations
  lark-drive.el                 Drive: dired-like browser, upload/download, owner names
  lark-sheets.el                Sheets: org-table read/write, create, find, export
  lark-contact.el               Contacts: user lookup, search, name resolution cache
  lark-meetings.el              Meetings: search, notes, recording with download links
  lark-whiteboard.el            Whiteboard: create and update whiteboards

ai/
  lark-ai.el                    AI: prompt -> plan -> execute -> synthesize -> present
  lark-ai-skills.el             AI: skill registry, loader, system prompt assembly
  lark-ai-context.el            AI: buffer context extraction for all domain modes

test/
  lark-*-test.el                ERT tests for each module
#+end_example

All CLI calls funnel through =lark--run-command= (async) or
=lark--run-command-sync=.  Output is =--format json= by default, parsed
with =json-parse-string=.  CLI debug lines (=[cmd +sub]= prefixes and
=tip= lines) are automatically stripped before JSON parsing.

=lark.el= automatically adds =core/=, =domain/=, and =ai/= to the
=load-path= at load time, so a single =(add-to-list 'load-path "/path/to/lark.el")=
is sufficient.
